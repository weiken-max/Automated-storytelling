# 自动讲故事视频项目：运作原理与执行流程详解（V2026.04.27）

> 适用项目：`game_video_clone_agent`  
> 文档目标：以当前代码为准，说明工业化 16 宫格视频生产管线如何稳定运行

---

## 1. 项目核心目标

这是一个“音频先行、时间锁死、画面后配、绝对卡点”的工业化叙事视频系统。

一句话：

1. 先产纯净长文案与定妆照；
2. 先把主音轨和物理时间轴锁死；
3. 再做剧情切分和触发点插值；
4. 最后批量生图并按绝对时间装配成片。

---

## 2. 全链路 Blueprint（当前实现）

### 阶段一：剧本与绝对时钟建立

#### Step 1：文案解耦与定妆（`src/story_planner_v6.py` + `src/ref_generator.py`）

- 输入：主题（topic）
- 处理：
  - 生成 `temp_synopsis.json`（梗概）
  - 生成 `full_story_v6.json`（1700 字纯净长文案）
  - 并行触发定妆照生成与阶段锚点回写
- 输出：
  - `data/scripts/temp_synopsis.json`
  - `data/scripts/full_story_v6.json`
  - `data/refs/*`（定妆照）

#### Step 2：音频先行与物理时间轴（`src/step1_writer_v6.py --phase phase2`）

- 输入：`full_story_v6.json` 中 `master_design.full_narration`
- 处理：
  - 调用 Edge-TTS 生成 `master.mp3`
  - 解析 VTT 生成真实微句时间轴 `master_srt.json`
  - 行号锚定后让 LLM 只做 Beat 索引切分
- 输出：
  - `data/audio/master.mp3`
  - `data/audio/master_srt.json`
  - `data/scripts/pseudo_srt.json`

---

### 阶段二：安全隔离的剧情拆解（Beat）

仍由 `src/step1_writer_v6.py --phase phase2` 完成：

- 代码给微句子临时加行号；
- LLM 仅返回 `start_index/end_index`；
- 代码据此回收出带真实时间的 Beat（`pseudo_srt.json`）；
- 行号只在中间过程存在，不进入持久产物。

---

### 阶段三：视觉规划与时间插值（Subshot）

#### Step 4：分镜与 trigger_time 计算（`src/step1_writer_v6.py --phase phase3`）

- 输入：`pseudo_srt.json` + `full_story_v6.json`
- 处理：
  1. LLM 先在 Beat 原文中按视觉转折点插入 `<subshot>` 标签，形成带标签文本；
  2. 代码搜索 `<subshot>` 在原文中的字符相对位置（百分比），并映射到对应 cue 的真实 `start_time/end_time` 做线性插值，计算每个分镜 `trigger_time`；
  3. 在生成 `visual_prompt` 时，必须同时输入：带标签文本 + Beat 的故事概括 `summary` + 定妆照锚点，避免上下文丢失；
- 输出：
  - `data/scripts/narrative_v6_final.json`

该文件核心结构如下：

```json
{
  "metadata": {},
  "timeline": [
    {
      "subshot_id": "S_001",
      "beat_id": "beat_1",
      "trigger_time": 0.0,
      "text_segment": "...",
      "visual_prompt": "..."
    }
  ]
}
```

---

### 阶段四：16宫格高并发视觉生产

#### Step 5：打包生图（`src/step2_comic_generator_v6.py`）

- 输入：`narrative_v6_final.json.timeline`
- 处理：
  - 每 16 条 `visual_prompt` 作为一个批次，要求模型在单张 `16:9` 画幅内生成 `4x4` 宫格；
  - 子分镜必须严格按 Prompt 顺序从左到右、从上到下排版（Panel 1 -> Panel 16）；
  - 生成 `grid_batch_XXX.png`；
  - 调用 `image_processor.py` 切成单镜 `S_XXX.png`；
- 输出：
  - `data/storyboards/grid_batch_XXX.png`
  - `data/storyboards/S_001.png ...`

#### Step 6：脚本裁切（`src/image_processor.py`）

- 输入：16 宫格大图
- 处理：
  - 物理等分；
  - OpenCV 轮廓识别与去白边；
  - 预留 `upscale_image()` 高清放大接口；
- 输出：单镜图 `S_XXX.png`

---

### 阶段五：组装成片

#### Step 8/9：绝对时间装配（`src/step3_assembler_v6.py`）

- 输入：
  - `data/scripts/narrative_v6_final.json`（`trigger_time`）
  - `data/audio/master.mp3`
  - `data/storyboards/S_XXX.png`
- 处理：
  - 每镜时长 = 下一镜 `trigger_time` - 当前镜 `trigger_time`
  - 最后一镜补到音频结尾
  - `duration <= 0` 时自动兜底 `0.1s`
  - 挂载主音轨渲染成片
- 输出：
  - `data/exports/final_masterpiece.mp4`

---

## 3. 当前关键容错机制（已落地）

1. `step2_comic_generator_v6.py`：某批 16 宫格失败时，自动补齐占位 `S_XXX.png`，保证分镜编号与 timeline 对齐；
2. `step3_assembler_v6.py`：若缺少某张 `S_XXX.png`，使用黑帧占位，不再跳过该时间槽位；
3. `step3_assembler_v6.py`：`duration <= 0` 自动兜底，避免渲染异常中断。

---

## 4. 关键数据接口对齐关系

1. `full_story_v6.json.master_design.full_narration`  
   -> `step1_writer_v6.py phase2` 读取并生成音频与物理时间轴。

2. `pseudo_srt.json.pseudo_srt[*].cues`  
   -> `step1_writer_v6.py phase3` 读取并计算 `trigger_time`。

3. `narrative_v6_final.json.timeline[*].visual_prompt`  
   -> `step2_comic_generator_v6.py` 读取并打包生图。

4. `narrative_v6_final.json.timeline[*].trigger_time`  
   -> `step3_assembler_v6.py` 读取并计算每镜绝对时长。

5. `storyboards/S_XXX.png` 命名序列  
   -> `step3_assembler_v6.py` 以 `S_{i+1:03d}.png` 精准取图。

---

## 5. 一句话业务总结

当前版本已经从“语义估时”升级为“物理时间轴驱动 + 代码插值卡点”，可在保证画面批量生产效率的同时，维持叙事节奏与音画对齐稳定性。
