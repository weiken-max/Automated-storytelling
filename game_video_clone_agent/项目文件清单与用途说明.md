## 2. `src/` 核心代码清单（按主链路）

### Phase 1：剧本与定妆照

- `src/story_planner_v6.py`  
  生成 `temp_synopsis.json` 与 `full_story_v6.json`，并触发定妆照流程。

- `src/ref_generator.py`  
  读取梗概/长文案，生成主角阶段定妆照，回写角色锚点信息。

### Phase 2/3：物理时间轴与分镜插值

- `src/step1_writer_v6.py`  
  - `--phase phase2`：生成 `master_voice.mp3`、`master_srt.json`、`pseudo_srt.json`（Beat 切分）；  
  - `--phase phase3`：生成 `narrative_v6_final.json`（含 `trigger_time`）。Phase3 对每个 Beat **两次 LLM 调用**：① 仅插入 `<subshot>` 打标签；② 对已切段生成 `visual_prompt`（带上一 Beat 的 `summary`）。单 Beat 最多整轮重试 5 次（`MAX_VISION_LLM_RETRIES`），失败则抛错退出，**不静默跳过 Beat**。

### Phase 4：16 宫格生产与组装

- `src/step2_comic_generator_v6.py`  
  16 宫格批量生图；单批最多重试 5 次（`STEP2_MAX_BATCH_RETRIES`）；失败按阶段记录并在五次失败后中止 Step2。

- `src/image_processor.py`  
  将 16 宫格**物理均分为 16 张子图**（不做去白边）；可选 Real-ESRGAN 放大单镜。

- `src/step3_assembler_v6.py`  
  按 `trigger_time` + `master_voice.mp3` 进行纯 FFmpeg 绝对时间装配。

### 基础设施与支持模块

- `src/style_config.py`：全局配置、路径、模型参数。
- `src/image_engine.py`：生图 API 封装（异步）。
- `src/grid_validators.py`：Step2 宫格准入校验。
- `src/visual_era_context.py`：视觉年代/风格上下文生成辅助模块。
- `src/project_vault.py`：项目金库备份。
- `src/run_context.py`：Run-ID 批次目录管理与当前批次指针。
- `src/model_presets.py`：模型与供应商预设。
- `src/pipeline_errors.py`：结构化错误模型。
- `src/audio_mastering.py`：音频处理工具（扩展模块）。
- `src/cleanup.py`：清理脚本。
- `src/watch_progress.py`：进度观察工具。
- `src/__init__.py`：包初始化。

## 3. `feishu/` 交互层（重构后）

### 入口与配置

- `feishu/hub.py`：新飞书主入口（~230 行，精简架构）。负责事件注册与分发，业务逻辑全部委托给子模块。
- `feishu/hub_old.py`：旧版 hub.py 完整备份（156KB）。出现问题时删掉 `hub.py`，将本文件改回 `hub.py` 即可秒级回滚。
- `feishu/config.py`：全局常量中心。集中管理：16 种状态常量、28 种 Action 类型、所有快捷词表（审批词/取消词/刷新词等）、LLM System Prompt、状态 → 人类可读映射、合法状态转移表、卡片模板色等。**修改任何硬编码只需改这一个文件**。

### 状态管理

- `feishu/session/session.py`：Session 数据模型。包含状态转移验证（`transition_to()` 强制执行 `ALLOWED_TRANSITIONS` 规则）、进度管理、时长读写。
- `feishu/session/session_store.py`：Session 持久化存储（SQLite `sessions` 表）。**单表替代**旧版 `state_mgr.py`（`flow_state` 表）+ 6 个散 JSON 文件（`synopsis_card_mid.txt`、`visual_card_mid.txt`、`storyboard_card_holder.json`、`last_error_context.json`、`synopsis_duration_draft.json`、`last_operator_open_id.json`）。

### 卡片子系统

- `feishu/cards/base_card.py`：卡片基类。提供 `build()` / `send()` / `patch()` / `send_or_patch()` 统一接口，以及 `make_button()` / `make_md()` / `make_hr()` / `make_img()` 辅助方法。
- `feishu/cards/idle_card.py`：空闲态卡片（选题盲盒）。
- `feishu/cards/synopsis_card.py`：大纲审批卡片（含时长选择器）。
- `feishu/cards/character_card.py`：定妆审批卡片（含单阶段/配角重画按钮）。
- `feishu/cards/storyboard_card.py`：分镜审核卡片（含逐批打回按钮）。
- `feishu/cards/progress_card.py`：生产进度卡片（运维控制台，含资产探针、队列信息、断点续传按钮）。
- `feishu/cards/factory.py`：卡片工厂。根据 Session 状态自动选择对应卡片类（`STATUS → CardClass` 映射）。

### 消息与按钮处理

- `feishu/handlers/registry.py`：Action 注册表。字典分发（`action_type → Action实例`），替代旧版 400 行 `if/elif` 链。新增按钮只需在 `actions/` 下新建一个 Action 类并注册。
- `feishu/handlers/message_router.py`：消息路由器。处理文字消息：关键词快速通道（审批/取消/刷新/帮助等词表）+ LLM 对话兜底（含 `[TRIGGER]` 指令解析兼容）。
- `feishu/handlers/actions/`：27 个 Action 处理器文件：
  - `confirm_synopsis.py`：大纲确认 + 修订请求
  - `confirm_character.py`：定妆确认
  - `confirm_storyboard.py`：分镜确认 + 批次打回
  - `cancel_project.py`：取消项目
  - `pause_resume.py`：暂停/恢复（新增）
  - `refresh_topics.py`：换一批选题
  - `duration.py`：时长 ±30 秒 / 预设
  - `revision.py`：重写/重画定妆
  - `ops.py`：运维（状态/重试/中止/错误恢复）
  - `misc.py`：杂项（重启/确认新项目/启动恢复）

### 管线编排

- `feishu/pipeline/bridge.py`：管线桥接模块。封装对旧版生产函数（`run_project_pipeline`、`run_synopsis_setup`、`run_visual_setup`、`retry_single_step` 等）的访问，为新的薄入口提供统一接口。未来理想状态是将所有函数直接实现在 `pipeline/` 下，逐步去桥接。

### 基础组件

- `feishu/feishu_mgr.py`：飞书消息/卡片/媒体收发封装。
- `feishu/ideator.py`：选题生成逻辑。
- `feishu/watchdog.py`：守护与自愈重启。
- `feishu/__init__.py`：包初始化。
- `feishu/state_mgr.py`：旧版状态管理（SQLite `flow_state` 表），与 `hub_old.py` 配套保留，重构后已被 `session/` 替代。
- `feishu/run_story_planner_with_mock.py`：飞书侧规划流程辅助入口。
- `feishu/run_pid_registry.json`：按 Run-ID 记录运行中子进程 PID（运行时文件）。
- `feishu/hub.pid`：运行期 PID 文件（临时）。
- `feishu/last_operator_open_id.json`：最近交互用户 ID（运行时文件，用于重启恢复推送）。
- `feishu/temp_synopsis.json`：飞书链路兼容梗概缓存（运行时文件）。
- `feishu/test_check.py`：回归测试脚本（104 项检查，每次改代码后建议运行）。

## 4. `data/` 运行目录说明（最新主链路）

`data/` 下各子目录**只把 `.gitkeep` 提交到 Git**；运行中产生的 JSON、图、音、视频等**默认不入库**（见本目录下 `.gitignore` 中 `data/` 规则）。克隆仓库后会得到完整空文件夹结构；本地素材仅供本机使用。

### `data/runs/`（主运行目录，强制）

- `data/runs/current_run.json`：当前激活批次指针。
- `data/runs/Run_YYYYMMDD_HHMMSS_xxx/`：单批次独立工作区，典型子目录：
  - `scripts/`
  - `audio/`
  - `storyboards/`
  - `output/`
  - `refs/`
  - `logs/`

说明：
- Step1/Step2/Step3 默认只读写当前批次目录；
- 断点续传与历史回放通过切换 `current_run.json` 完成；
- 旧 `data/scripts|audio|storyboards|output` 口径已迁移为 Run-ID 目录口径。

### `data/scripts/`（兼容历史）

- `current_project.json`：当前项目标识。
- `temp_synopsis.json`：梗概审批稿（Phase 1）。
- `full_story_v6.json`：长文案与角色设计主数据（Phase 1）。
- `pseudo_srt.json`：Beat 拆分结果（Phase 2）。
- `narrative_v6_final.json`：最终分镜时间线（Phase 3）。

### `data/audio/`

- `master.mp3`：主音轨（历史命名；当前主链路推荐 `master_voice.mp3`）。
- `master_srt.json`：真实物理微句时间轴（Phase 2）。

### `data/storyboards/`

- `grid_batch_XXX.png`：16 宫格中间图（Phase 4-A）。
- `S_001.png ...`：切图后的单镜文件（Phase 4-A）。

### `data/exports/`（兼容历史）

- `final_masterpiece.mp4`：最终视频（历史命名；当前主链路推荐 `narrative_v6_final_epic.mp4`）。

### `data/refs/`

- 主角/配角/场景参考图与描述（定妆照资产池）。

### `data/anchors/`

- 历史项目金库存档（可追溯，不是主链路硬依赖）。

### `data/archive/`

- 清场时归档的旧版参考图或描述文件。

### `data/logs/`

- 当前主链路建议优先看 `data/runs/<run_id>/logs/`（Run 级日志）。
- Step2 关键文件：
  - `step2_failures.jsonl`：失败明细（含 stage/attempt/batch）
  - `step2_report.json`：Step2 汇总与 Step3 准入依据

### `data/output/`、`data/temp/`、`data/cache/`

- 若代码或脚本使用这些目录作中间/缓存/导出，**同样只保留 `.gitkeep` 在仓库中**，内容本地生成、不同步。

### `feishu/state.db`

- SQLite 数据库文件，包含：
  - `sessions` 表：所有用户会话状态（替代旧 `flow_state` 表 + 散 JSON 文件）
  - `chat_history` 表：LLM 对话历史
  - 旧 `flow_state` 表：迁移过渡期保留

## 5. 已落地的关键容错（与当前代码一致）

- `step2_comic_generator_v6.py`：按批次最多重试 5 次；连续失败会中止 Step2，不再补齐占位图。
- `step2_comic_generator_v6.py`：失败按阶段分类（IMAGE_GEN_API/IMAGE_DECODE/GRID_CROP/UPSCALE）写入 `step2_failures.jsonl`。
- `grid_validators.py`：Step2 默认采用"全图寻峰 + 结构校验"减少误杀。
- `feishu/hub.py`：Step2 子进程非 0 退出或门禁未通过时，向飞书推送明确失败通知。
- `step1_writer_v6.py`（Phase3）：Beat 分镜失败不静默跳过，整 Beat 重试耗尽后进程退出。
- `step3_assembler_v6.py`：纯 FFmpeg 合成链路（临时片段目录 + 自动清理）；缺失图用黑帧占位。
- `image_engine.py`：标准 async/await + Semaphore + 超时重试。
- `project_vault.py`：软复制容错，缺失文件仅 warning，不阻断后续备份。
- `feishu/hub.py`：全局 FIFO 队列、状态卡原地刷新、Run-ID 精准中止、`/status|/switch|/retry` 运维指令。
- `feishu/config.py`：状态转移 `ALLOWED_TRANSITIONS` 强制执行，运行时非法转移会被 `ValueError` 拦截。
- `feishu/session/`：SQLite 统一持久化，重启后自动恢复。
