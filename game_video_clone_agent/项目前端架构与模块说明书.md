## 🛠️ 各模块代码功能详解

### 1. `palette_studio.html` (骨架层)
*   **作用**：定义整个软件界面的大体框架（如左侧大纲编辑区、右侧渲染区、各个弹窗容器）。
*   **改动后状态**：删除了所有 inline 样式与 JS 逻辑。这使得该文件极为清晰，不再会有 ID 重复、标记闭合错误等致命 HTML 语法问题。

### 2. `css/studio.css` (表现层)
*   `.hover-shake-bubble`：调色盘色块物理气泡振颤动画。
*   `.space-splash` 与 `.splash-circle`：频道激活时的空间吞没扩散动画特效。
*   `.glass-studio-panel`：高品质哑光毛玻璃效果面板。
*   **作用**：负责将所有定制的 CSS 类名集中化管理，确保软件界面维持电影级的设计感。

### 3. `js/presets.js` (配置层)
*   `DEFAULT_PRESETS`：出厂的大模型系统提示词（如画面锚点、解说旁白文案、行业潜规则等模板）。
*   `DEFAULT_CHANNELS`：内置原厂频道数据（目前含“剧情故事”和“硬核科普”）。
*   `loadChannelsFromStorage()` / `saveChannelsToStorage()`：负责本地浏览器缓存的读取与保存，支持从 Python 后端拉取物理硬盘备份。

### 4. `js/bridge.js` (通信层)
*   `checkApiConnection()`：通过周期性心跳点亮顶部的“LIVE (BRIDGE CONNECTED)”绿灯。
*   `toggleBackstageDrawer()`：拉出或收起“后台大盘实时审计面板”。
*   `detectRestorableSession()`：检查用户上次意外退出时保留的进度断点。
*   `restoreSession()`：一键将界面及内存状态（大纲、定妆大卡、视频路径）瞬间自愈恢复。

### 5. `js/app.js` (核心逻辑层)
*   `const state = { ... }`：核心状态机，记录当前在哪一步骤、哪个频道激活、历史对比图状态等。
*   `renderChannelGrid()`：在首页绘制自定义与出厂频道入口。
*   `applyD2Feedback()`：用户输入修改意见（例如“增加猫咪助手”）后，请求大模型重写大纲。
*   `syncRawScriptToState()`：动态计算 JSON 报文预览器。

## 💡 后续开发与 AI 协同修改指南
当您在未来的开发中，指示 AI 助手修改系统或增加功能时，请让它遵守如下原则，这样可以最大程度**节省 token** 并**防止代码改错**：

*   **如果需要调整 UI 配色、字体、动画**：
    > 💬 *指令示例：请修改 `css/studio.css` 中的毛玻璃背景模糊度和配色。*
*   **如果需要增加大模型提示词模板的初始字段或修改系统预设**：
    > 💬 *指令示例：请在 `js/presets.js` 的 `DEFAULT_PRESETS.outline` 中加入新的结构化输出限制。*
*   **如果需要对接新的 Python 后端交互接口**：
    > 💬 *指令示例：请在 `js/bridge.js` 中增加一个新的接口请求方法以调用后端的某功能，并在 `js/app.js` 里关联按钮点击事件。*
*   **如果需要修改或新增某个按钮的交互逻辑、弹窗控制**：
    > 💬 *指令示例：请打开 `js/app.js`，修改某某交互函数的触发逻辑。*
