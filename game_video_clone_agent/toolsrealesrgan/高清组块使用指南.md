# 高清组块使用指南

本指南说明 `game_video_clone_agent/toolsrealesrgan` 目录（Real-ESRGAN ncnn Vulkan 便携版）的**用途、文件含义、命令行用法**，以及如何做一次**快速效果验证**。  
裁切后每张 `S_XXX.png` 会由 `src/image_processor.py` 的 **`upscale_image()`** 调用本目录 **`realesrgan-ncnn-vulkan.exe`** 覆盖保存（可用环境变量关闭或调参，见该文件）。亦可单独用命令行对比效果。

### 当前用的是哪个模型？

- **项目里没有单独的「高清 Python 脚本」**：现阶段是靠 **`realesrgan-ncnn-vulkan.exe` 命令行**完成推理。
- **不写 `-n` 时**：程序使用内置默认值 **`realesr-animevideov3`**（与上游 `README_windows.md` 一致）。你此前生成的 **`test_output.png`** 若按文档示例未加 `-n`，即为该模型。
- **换模型**：必须通过 **`-n 模型名`** 指定；动漫视频向模型 **`realesr-animevideov3`** 还需配合 **`-s`（2 / 3 / 4）** 选用 `models` 里对应的 `x2` / `x3` / `x4` 权重。另两个模型为 **4× 专用**，见下表。

本目录 `models` 与标准 Windows 发行包一致时，共 **3 种逻辑模型**（不是 3 个文件，而是 3 组：其中 `realesr-animevideov3` 占 3 个放大档位）：

| `-n` 模型名 | 典型 `-s` | `models` 中主要文件（成对 `.param` + `.bin`） | 适用场景（经验向） |
|-------------|-----------|-----------------------------------------------|-------------------|
| `realesr-animevideov3` | **2、3 或 4**（需与档位一致） | `realesr-animevideov3-x2`、`x3`、`x4` | 动漫视频、番剧截图、运动漫画格；可调倍率 |
| `realesrgan-x4plus` | 一般为 **4** | `realesrgan-x4plus` | 照片、写实场景、通用非二次元 |
| `realesrgan-x4plus-anime` | 一般为 **4** | `realesrgan-x4plus-anime` | 插画、平面漫画、二次元静图 |

## 1. 与本项目的关系

| 环节 | 说明 |
|------|------|
| **裁切** | `process_16_grid()` 把 4×4 宫格切成子图并去白边，保存为 `S_XXX.png`。 |
| **高清** | 每保存一张子图会调用 `upscale_image(out_path)`，内部执行 **`realesrgan-ncnn-vulkan.exe`** 超分并覆盖原文件（见 `src/image_processor.py` 与环境变量说明）。 |
| **本目录** | 放置 **Real-ESRGAN ncnn-vulkan** 可执行文件与 **models**；主流程与命令行测试共用此路径。 |

## 2. 目录结构与文件含义

```
toolsrealesrgan/
├── realesrgan-ncnn-vulkan.exe   # 主程序（官方发行包内；若本仓库未包含则需自行下载）
├── README_windows.md            # 上游精简说明（命令摘录）
├── 高清组块使用指南.md          # 本文档
└── models/
    ├── *.param                  # ncnn 网络结构定义（文本，描述各层）
    ├── *.bin                    # 模型权重（二进制；与 .param 同名配对）
    └── …
```

### 2.1 可执行文件

- **`realesrgan-ncnn-vulkan.exe`**：基于 Vulkan 的推理程序，**不需要**本机安装 CUDA / PyTorch。  
- 若当前文件夹里**没有**该 exe：请到 [Real-ESRGAN Releases](https://github.com/xinntao/Real-ESRGAN/releases) 下载带 **ncnn-vulkan**、**windows** 字样的压缩包，解压后将 `realesrgan-ncnn-vulkan.exe` 与完整 `models` 目录放到本路径（或与文档中的路径一致）。

### 2.2 `models` 下常见文件

| 类型 | 含义 |
|------|------|
| **`*.param`** | ncnn 模型结构（可读文本），与同名 **`*.bin`** 一起使用。 |
| **`*.bin`** | 模型权重，体积较大，常与 `.param` 成对出现。 |

**与 `-n` / `-s` 的对应关系（便于对照目录里的文件名）：**

- **`realesr-animevideov3`**：`models` 内有 **三套** `realesr-animevideov3-x2`、`x3`、`x4`。命令行写 **`-n realesr-animevideov3 -s 2`**（或 `3`、`4`）时，exe 会加载对应 **x2 / x3 / x4** 那一套。
- **`realesrgan-x4plus`**：对应 **`realesrgan-x4plus.param` / `.bin`**，通常配合 **`-s 4`**。
- **`realesrgan-x4plus-anime`**：对应 **`realesrgan-x4plus-anime.param` / `.bin`**，通常配合 **`-s 4`**。

若只见到 `.param` 没有 `.bin`，说明权重未放全，需使用官方完整 **windows ncnn-vulkan** 压缩包补全。

实际推理时由 exe 根据 `-n`、`-s` 等参数选择模型；**缺 `.bin` 或 exe 时无法运行**。

### 2.3 其它说明文件

- **`README_windows.md`**：官方风格的命令示例（单图、文件夹批处理、动漫视频抽帧→放大→合成流程）。

## 3. 环境准备

1. **显卡驱动**：安装较新的 **Vulkan** 可用驱动（多数独立显卡 / 较新核显均可尝试）。  
2. **完整工具包**：确保本目录下同时存在 **`realesrgan-ncnn-vulkan.exe`** 与 **`models`** 内成对的 **`.param` + `.bin`**。  
3. **可选**：若要做视频测试，需已安装 **`ffmpeg`** 并可在命令行中调用。

## 4. 使用方法（命令行）

在 PowerShell / CMD 中先进入本目录，**勿使用机器相关的绝对路径**。任选其一：

- 当前工作目录已是仓库里的 **`game_video_clone_agent`**：`cd .\toolsrealesrgan`
- 当前工作目录是**仓库根目录**：`cd .\game_video_clone_agent\toolsrealesrgan`

下文示例均假定当前目录已为 **`toolsrealesrgan`**（若已用资源管理器在该文件夹打开终端，可跳过 `cd`）。

### 4.1 单张图片（最快验证）

在**仓库根目录**下进入本目录并运行（相对路径，换机仍有效）：

```powershell
cd .\game_video_clone_agent\toolsrealesrgan
.\realesrgan-ncnn-vulkan.exe -i test_input.png -o test_output.png
```

若终端当前目录已是 **`game_video_clone_agent`**，将第一行改为 `cd .\toolsrealesrgan`；若已用资源管理器在 `toolsrealesrgan` 中打开终端，可省略 `cd`，只运行第二行。

**不写 `-n` 时等于使用默认模型 `realesr-animevideov3`**（若同时未写 `-s`，程序仍按内置默认倍率处理；为与 `models` 中 x2/x3/x4 档位严格对应，建议显式写上 `-s 2`、`3` 或 `4`）。

### 4.1.1 三种模型各试一遍（推荐）

在 **`toolsrealesrgan`** 目录下，将输入文件名换成你的图（示例为 `test_input.png.png`），分别输出到不同文件，方便并排对比：

```powershell
# ① 动漫视频向（默认家族）：4× 档位示例
.\realesrgan-ncnn-vulkan.exe -i "test_input.png.png" -o "out_animevideov3_s4.png" -n realesr-animevideov3 -s 4

# ② 通用写实 / 照片向：一般为 4×
.\realesrgan-ncnn-vulkan.exe -i "test_input.png.png" -o "out_x4plus.png" -n realesrgan-x4plus -s 4

# ③ 二次元插画向：一般为 4×
.\realesrgan-ncnn-vulkan.exe -i "test_input.png.png" -o "out_x4plus_anime.png" -n realesrgan-x4plus-anime -s 4
```

同一 **`realesr-animevideov3`** 还可改 **`-s 2`** 或 **`-s 3`**，对应 `models` 里 **x2 / x3** 权重，输出文件名可自行改为 `out_animevideov3_s2.png` 等。

其它指定模型名示例（单图）：

```powershell
.\realesrgan-ncnn-vulkan.exe -i test_input.png -o test_output.png -n realesr-animevideov3 -s 4
.\realesrgan-ncnn-vulkan.exe -i test_input.png -o test_output.png -n realesrgan-x4plus -s 4
.\realesrgan-ncnn-vulkan.exe -i test_input.png -o test_output.png -n realesrgan-x4plus-anime -s 4
```

### 4.2 常用参数（摘）

| 参数 | 含义 |
|------|------|
| `-i` | 输入文件或输入**文件夹** |
| `-o` | 输出文件或输出**文件夹** |
| `-n` | 模型名称（如 `realesr-animevideov3`、`realesrgan-x4plus`、`realesrgan-x4plus-anime`） |
| `-s` | 放大倍数。**`realesr-animevideov3`** 常用 `2`/`3`/`4`（对应 `models` 中 x2/x3/x4）；**`realesrgan-x4plus`** / **`realesrgan-x4plus-anime`** 一般为 `4` |
| `-f` | 批量输出格式，如 `jpg`、`png` |

### 4.3 文件夹批处理示例

（需事先建好输出目录，例如 `out_jpg`。）

```powershell
.\realesrgan-ncnn-vulkan.exe -i .\input_folder -o .\out_jpg -n realesr-animevideov3 -s 2 -f jpg
```

更多示例见 **`README_windows.md`**（含动漫视频：ffmpeg 抽帧 → RealESRGAN → 再合成视频）。

### 4.4 使用注意

- 大图会被切成多块（tile）处理再拼接，极端情况下可能出现**块状接缝**或与 PyTorch 版略有差异（上游说明亦提及）。  
- 动画/漫画截图优先试 **realesr-animevideov3** 或 **realesrgan-x4plus-anime**；照片或写实画面可试 **realesrgan-x4plus**。

## 5. 初步测试指南（建议按顺序做）

目标：用**最少步骤**确认「能否跑通」和「哪种模型更适合你的素材」。

### 步骤 A：准备一张测试图

1. 在项目内任意位置放一张 **PNG 或 JPG**（建议不要太小也不要过大：例如长边 512～2048 像素）。  
2. 可复制到 `toolsrealesrgan` 下并命名为 `test_input.png`，便于照抄命令。

### 步骤 B：确认程序可用

1. 打开终端，`cd` 到 **`toolsrealesrgan`**。  
2. 执行 **4.1** 中单张图片命令。  
3. **成功标志**：生成 `test_output.png`（或你指定的输出文件），无报错退出。

若报错「找不到 exe」：补全官方发行包中的 **`realesrgan-ncnn-vulkan.exe`**。  
若报错与模型相关：检查 **`models`** 下对应 **`.bin`** 是否存在且与发行包一致。

### 步骤 C：对比不同模型（肉眼选效果）

对**同一张**测试图，分别输出到不同文件。可直接复制 **4.1.1 节「三种模型各试一遍」**中的三条命令；若使用 **`realesr-animevideov3`**，建议至少试 **`-s 2`、`3`、`4`** 各一张，再与 **`realesrgan-x4plus`**、**`realesrgan-x4plus-anime`**（通常 `-s 4`）对比。

在同一显示器上放大查看：线条是否糊、是否有蜡像感、文字是否发糊，据此选定后续流水线默认模型与默认 `-n` / `-s`。

### 步骤 D：记录结论（便于后续接代码）

建议简单记下：

- 本机是否一键跑通；  
- 素材类型（游戏截图 / 漫画 / 照片）；  
- 选定模型名与 `-s`；  
- 可接受的耗时（单张耗时心中有底即可）。

### 步骤 E（可选）：与主流程“对接预期”

主流程已在 **`src/image_processor.py` → `upscale_image()`** 内调用 **`toolsrealesrgan/realesrgan-ncnn-vulkan.exe`**（临时输出再覆盖原图）。若需改默认模型、倍数或显卡索引，请设置环境变量 **`REALESRGAN_MODEL` / `REALESRGAN_SCALE` / `REALESRGAN_GPU`**，或 **`REALESRGAN_DISABLE=1`** 关闭放大；详见该源码文件顶部的函数注释。
